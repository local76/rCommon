# Security Policy

We take the security of `library` and the entire `local76` ecosystem seriously. If you believe you have found a security vulnerability, please report it immediately so that we can address it.

## Reporting a Vulnerability

Please do **not** report security vulnerabilities via public GitHub issues. Instead, report them privately using one of the following methods:

- Email the maintainer directly at: [security@local76.org](mailto:security@local76.org)
- Submit a report via the GitHub Security Advisory tab on the repository (if available).

Please include the following details in your report:
- A description of the vulnerability and its potential impact.
- Step-by-step instructions (or a proof-of-concept script) to reproduce the issue.
- The version of `library` and target operating system where the issue was observed.

## Scope

This security policy applies to the main `library` library and its official packaging/distribution scripts.

## Response & Disclosure Process

1. **Acknowledgment**: We will acknowledge receipt of your report within 48 hours.
2. **Investigation**: We will investigate the issue and coordinate a fix.
3. **Patching**: We aim to release a patched version of `library` as soon as a resolution is verified.
4. **Advisory**: Once fixed, a security advisory will be published, and you will be credited for the discovery unless you request to remain anonymous.
